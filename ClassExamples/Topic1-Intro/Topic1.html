3.       Components and Variables: 

o   Two scripts:

      • MoveInX: behavior is simple
        • Bounces in X
        • Allow changing of bouncing speed (Inc/Dec Delta)
      • MainControl: Two references to components
        • An object: the moving cube (Object)
        • A script: the moving component on the cube
          • Pay attention to this reference!
          • This is a reference to a component, but, MainControl is NOT hosting the component!
          • This reference:
            • Does not cause Unity to call Awake/Start/Update
            • The transform/gameObject references from this component are NOT the same as those of MainControl
    • Three buttons:
      • DeleteMoveInX: delete the moving script (component)
      • CreateCube + MoveInX: creates a MoveInX for the cube (what happens when a cube has two MoveInX?
      • DeleteCube: deletes the cube
    • Run the game: Observe the Cube, its speed alternates (controlled by MainControl). Try the following: 
      • Select the CubeHasMoveComponent from the Hierarchy and examine its components in the Inspector
      • Click on DeleteCube'sMove, what happened?
      • Click on CreateCuve + MoveInX, what happens? What happens when you click on this multiple times?
      • Click on DeleteCube, and click on CreateCube again, what happens now? (A new Cube is created)

    • First to note: "C# Scripts" (MonoBehavior objects) are "components" of a GameObject, Scripts, or MonoBehaviors are not GameObject themselves.
      • Notice: how these things are Destroy() [same function call to remove them]

    • Second to note: components can be referenced by variables, created/deleted dynamically at run time, and, we can create many copies of component to be attached to the same GameObject.
      • Notice: both GameObject and "C# Scripts" can be created and deleted during run time.
      • The action of "dragging" a script onto a GameObject to associate the defined behavior with the object, actually:
        • Create a new instance of the script object (e.g., MoveInX)
        • Create a reference from the GameObject to the new script (e.g., the MoveInX object.)
        • This is exactly the same as calling GameObject.AddComponent<DataType>() [called in MainControl::CreateComponent()]. 
      • Keep on clicking the "Create MoveInX" button to attach multiple copies of the component on the Cube. Similarly, in the editor, you can drag the script onto the Object multiple times.

Make sure you understand: the difference between when a class is defined and when the class is instantiated.

  • Defined: when you typed in the code for a MonoBehavior object, and the code compiles where you can run the game.
  • Instantiated: when a GameObject with the defined MonoBehavior object is created and shows up in the Scene Hierarchy.
  • Instantiated:When a MonoBehavior script is dragged into a GameObject that is in the world/Hierarchy
  • NOT Instantiated:When a Prefab with the MonoBehavior script is created (in the Resource folder), NO NEW instances are created
  • Instantiated: When this PreFab is created.
