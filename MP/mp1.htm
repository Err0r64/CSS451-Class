 Approach

Please refer to my attempt at the solution (WebGL build, simply click and run). Initially, your world consists of a CreationPlane and a CreationTarget:

  • CreationPlane: is a Quad with
    • Transform:
      • Position: 0, 0, 0
      • Rotation: 90, 0, 0 (such that the Quad is on the x/z plane)
      • Scale: 14, 14, 14
    • Note: the corners of the plane spans between –7 to +7
    • If you choose to work with a Plane, your transform settings will be different, but, it is important to make sure the size and location of the CreationPlane to be specified as above.
  • CreationTarget: is a sphere with
    • Transform:
      • Position: user defined!
      • Rotation: 0, 0, 0
      • Scale: 0.1, 0.1, 0.1 0.5, 0.5, 0.5
    • Color: black

You must also support a radio button system that allows the creation of a Sphere, Cylinder, or Cube. Here are the specifications for each of the shape:

  • Sphere: (not circular so we can observe the rotation)
    • Scale= (0.5, 1, 1)
    • Speed = 1 unit per second, along the y-axis
    • Rotation about the y-axis: 45-degree per second
    • Range: 0 < y < 5
    • Color:
      • When traveling in the positive-Y: color= (1, 1, 1)
      • When traveling in the negative-Y: color=(1, 0, 1)
  • Cylinder: (not symmetrical so we can observe the rotation)
    • Scale = (0.5, 1, 1)
    • Speed = 1.5 unit per second, along the z-axis
    • Rotation speed = about y-axis, 90-degrees per second
    • Range: 0 < z < 5
    • Color:
      • When traveling in the positive-Z: color=(1, 1, 1)
      • When traveling in the negative-Z: color=(0, 1, 1)
  • Cube:
    • Size of 1 (scale = 1)
    • Speed = 2 unit per second, along the x-axis
    • Rotation speed = about y-axis, 135-degrees per second
    • Range: 0 < x < 5
    • Color:
      • When traveling in the positive-X: color=(1, 1, 1)
      • When traveling in the negative-X: color=(1, 1, 0)

Behaviors

There are three basic behaviors that you must support.

1)       Primitive Created: This is a radio button where at most one of the check-marks can be checked.

a)       When user click on a primitive type, a new primitive with the corresponding type is created at the CreationTarget and the check mark stays.

b)       Clicking on the same primitive type consecutively the second time will un-check and not create a new primitive.

c)       When user click to delete a primitive, all check marks must be off.

d)       In all cases, a newly created shape should be resting on the CreationPlane and attempt to travel according to the above definition. If the creation position is larger than the valid range of 5, the created shape will then travel in the negative direction. After a shape’s first change of traveling direction, it should never cross the 0 to 5 range.

2)       Creation target manipulation: You must support Left-Mouse-Button click (LMB):

a)       When LMB is over the CreationPlane and not over any other objects, the CreationTarget is moved to the mouse click position.

b)       Note: CreattionTarget must always be entirely above the CreationPlane.

3)       Object deletion: LMB over a created object should delete the object. Note: one can never delete the CreationPlane or the CreationTarget.

 Hints

  1. My MainCamera is located at (0, 8, -16), with rotation (23, 0, 0). FOV of 45.
    • Learn the trick: in the Hierarchy Window, select the MainCamera, RMBàAlignWithView and see what happens!
  2. For radio buttons, I used Toggle and ToggleGroup. Creating ToggleGroup is slightly tricky, you will select the GameObject, and then, Inspector, click on the Add Component àUI.
  3. I googled “unity elapsed time” to try to find out how to work with per-second movements and rotations. Take a look at deltaTime (https://docs.unity3d.com/ScriptReference/Time-deltaTime.html),
  4. To avoid mouse-click selecting the CreationTarget, I place the corresponding sphere into a different Layer, and Raycast(ray, hitInfo, layer-1) [the default layer] to click select floor or the created objects.
    • If mouse click is on the floor, I move the CreationTarget over to the (x, someValue, z) where x, and z are from the mouse-click.
    • Do a google on “unity layer” to find out more about Layers.
  5. Take a look at Rotate(Axis, degree) to rotate a GameObject
